# 🕊️ Asociación Civil Misioneros Unidos de Isidro Casanova

Sitio web institucional diseñado para fortalecer la presencia digital de la organización, generar confianza en los visitantes y, principalmente, incentivar las donaciones para sostener el trabajo social y misionero.

## 🎯 Objetivos del Proyecto
- **Generar Confianza**: Mostrar la trayectoria y autoridad de la asociación desde 2011.
- **Incentivar Donaciones**: Facilitar los medios de aporte mediante secciones claras de impacto y transparencia.
- **Visibilidad de Impacto**: Exponer las misiones y ayuda social a través de contenido multimedia.
- **Canal de Contacto**: Optimizar la comunicación entre la comunidad, voluntarios y donantes.

## 🚀 Características Principales
- **Arquitectura Modular**: Estructura de archivos organizada para facilitar el mantenimiento y escalabilidad.
- **Diseño Mobile-First**: Adaptación completa a dispositivos móviles, tablets y computadoras de escritorio.
- **Optimización de Carga**: Uso de HTML/CSS/JS puro para garantizar la máxima velocidad de respuesta.
- **SEO Optimizado**: Implementación de etiquetas semánticas y meta-datos para mejorar la visibilidad en buscadores.
- **Secciones Especializadas**:
    - **Home**: Hero emocional con CTA directo a donaciones y video de impacto.
    - **Nosotros**: Historia institucional, Visión, Misión y biografía del fundador Jonatan Pereda.
    - **Media**: Galería responsiva de misiones y videos integrados de YouTube.
    - **Donaciones**: Desglose transparente del uso de fondos y métodos de pago simulados.
    - **Contacto**: Formulario validado con JS y datos de ubicación física.

## 🛠️ Stack Tecnológico
El proyecto ha sido desarrollado utilizando tecnologías web estándar para asegurar la máxima compatibilidad y velocidad:
- **HTML5**: Estructura semántica para accesibilidad y SEO.
- **CSS3**: Estilos modernos con variables, Flexbox y CSS Grid.
- **JavaScript (Vanilla)**: Interactividad ligera para el menú móvil y validación de formularios.

## 🌐 Despliegue (Deployment)
Este proyecto está optimizado para ser desplegado en **GitHub Pages**. 

1. Sube el código a un repositorio de GitHub.
2. Ve a **Settings** > **Pages**.
3. Selecciona la rama `main` (o la rama donde esté el código) y la carpeta `/ (root)`.
4. Guarda los cambios y el sitio estará disponible en `https://tu-usuario.github.io/nombre-del-repo/`.
